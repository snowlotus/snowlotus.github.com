学习笔记-js中难点梳理

2019-06-18

js的语言特性导致有很多知识点在日常开发中容易混淆或者难理解的，在此通过代码示例和文字总结加深理解，同时也方便后续复习查阅

this

与其它语言相比，函数的this关键字在javascript中表现略不同。此外，在严格模式、非严格模式及es2015的箭头函数中也各有不同。

一般情况下（箭头函数除外）,this的指向与声明位置无关，与调用位置有关


此时调用showName方法的是obj，故this指向obj


1. 在anotherObj的作用域中调用的tempFunc，tempFunc中的this无明确指向，故指向window
2. 在anonymousObj的作用域中执行匿名函数，this也是指向window


apply可以改变this的指向为第一个参数
故在obj.showName.apply()中this指向空，即window
在obj.showName.apply(anotherObj)中this指向空anotherObj


在es6中，箭头函数（又称lamdba表达式），是在声明的时候绑定this的而非调用时


在严格模式和非严格模式下，this的指向还略有不同
从上述结果可知，严格模式下，this没有明确指向时,指向undefinde而非window

从上述代码片段及运行结果可得知，this有以下特性（坑）

  • this 的指向和调用位置有关，与声明位置无关
  • 没有明确指向的时候，this指向window
  • 在浏览器环境下，setTimeout、setInterval和匿名函数执行时this的指向为全局对象window
  • call和apply能够强制改变this的指向为当前的第一个传参
  • 在es6中，箭头函数（又称lamdba表达式），是在声明的时候绑定this的
  • 严格模式下，this没有明确指向时,指向undefinde而非window

作用域

js中的作用域也是一个非常重要和易出错的地方，话不多说，先上代码


上面的代码在执行时别js引擎解析成如下的代码
function foo(){ var a;//undefined console.log(a); a=2; } foo();


上面代码等价与下面
var show;//undefined show (); show=function (){ console.log(1) }


上面代码等价于以下代码
function show(){
console.log(‘show’)
}
var show=11;
typeof show;
故由此可知，函数的变量提升优于变量


虽然此处有if,但js中没有块级作用域的概念，变量someVar依然也会发生作用域提升。

从上述代码片段及运行结果可得知，js的作用域有以下特性

  • 函数和变量的声明都会产生作用域提升
  • 变量的上升只是声明会提升，赋值不会
  • 函数表达式（包括匿名和具名）不会发生作用域提升
  • 函数提升的优先级高于变量，且不会被同名变量覆盖
  • 声明的提升不会被条件判断给控制住（不过不同浏览器的实现不同，有的会控制有的不会控制）
